# Phase 4 — Power BI Executive Dashboard

## Overview

An interactive three-page Power BI dashboard that communicates actuarial risk
insights from the Python model to non-technical stakeholders. Scenario slicers
allow real-time switching between Base, Optimistic, and Stress cases. All data
is sourced from CSV files exported by the Python pipeline.

---

## Dashboard Pages

### Page 1 — Executive Summary
High-level portfolio overview for senior management and board-level reporting:

- Combined Ratio and Loss Ratio gauges with benchmark thresholds
- Net Profit and premium adequacy status
- Solvency Ratio with 1.0× regulatory minimum reference line
- Ruin Probability (net of reinsurance)
- Scenario comparison bar chart (Base / Optimistic / Stress)
- Automated risk status cards (Profitable / Solvent / Pricing flags)

### Page 2 — Tail Risk Analysis
Deep-dive into capital requirements and reinsurance effectiveness:

- Risk metrics table (Mean Loss, VaR 95%, VaR 99%, VaR 99.5%, ES 99%)
- Solvency gauge vs 1.0× regulatory minimum and 1.5× management target
- VaR escalation chart (95% → 99% → 99.5%)
- Monte Carlo loss distribution histogram (100,000 simulations)
- Reinsurance impact panel: Gross VaR vs Net VaR, treaty parameters

### Page 3 — Portfolio Analytics
Granular portfolio composition and claims behaviour:

- Product mix (Motor, Property, Liability) — premium and policy count
- Claims frequency heatmap by product and region
- Claims mix: Attritional vs Large vs Catastrophe
- Top claims ranked by gross loss with XL recovery analysis
- Underwriting year development — Loss Ratio, Paid-to-Incurred, IBNR trend

## Data Sources

CSV files in `phase4_powerbi/power_bi/` are generated by `export_model_results.py`:

| File                       | Contents                                    |
|----------------------------|---------------------------------------------|
| `model_results.csv`        | All Python model outputs (VaR, ES, SCR, etc)|
| `policies.csv`             | Policy-level data from SQLite database      |
| `claims.csv`               | Claims data with XL recovery calculations   |
| `underwriting_years.csv`   | IBNR and development data by year/product   |
| `reinsurance_treaties.csv` | Treaty parameters for dashboard reference   |

## How to Use

```bash
# Step 1: Regenerate the source CSV files from the Python model
python phase4_powerbi/power_bi/export_model_results.py

# Step 2: Open the dashboard
# Open phase4_powerbi/power_bi_insurance_risk_dashboard.pbix
# in Power BI Desktop

# Step 3: Refresh data if prompted
# Home → Refresh
```

> **Requirement:** Power BI Desktop (free download from Microsoft).
> The `.pbix` file is not viewable in the browser version of Power BI
> without a Pro licence.

## Key Design Decisions

- **Scenario slicer** drives all visuals dynamically — single source of truth
- All monetary values displayed in **₹ Crore** for executive readability
- **Red/amber/green** conditional formatting on all ratio metrics
- Solvency gauge shows both **1.0× regulatory** and **1.5× management** thresholds
- Loss distribution chart uses **bin boundaries aligned to VaR percentiles**
  so risk thresholds are visually apparent in the histogram
